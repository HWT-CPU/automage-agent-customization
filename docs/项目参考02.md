# 项目参考 02：LangGraph Supervisor

## 项目链接

https://github.com/langchain-ai/langgraph-supervisor-py

## 项目简介

`langgraph-supervisor-py` 是 LangChain / LangGraph 生态中的一个开源项目，用于构建分层式多 Agent 系统。

它的核心思想是：由一个 Supervisor Agent 负责调度多个 Worker Agent；同时也支持更复杂的多层结构，即一个顶层 Supervisor 管理多个子 Supervisor，每个子 Supervisor 再管理各自的 Worker Agent。

这种结构和 AutoMage-2 的三级 AI 架构非常接近。

AutoMage-2 的目标结构是：

- 公司领导层 Agent：顶层决策节点
- 部门领导层 Agent：中间汇总与分发节点
- 岗位级 Agent：一线执行节点

LangGraph Supervisor 可参考的结构是：

- Top-level Supervisor：顶层调度者
- Team Supervisor：子团队管理者
- Worker Agents：具体执行者

## 可以作为我们项目开发参考的内容

### 1. 三级 Agent 架构组织方式

AutoMage-2 需要实现三级架构：

- Staff Agent：负责填报日报、获取任务、上报执行数据
- Manager Agent：负责读取下属数据、生成部门汇总、识别风险、分发任务
- Executive Agent：负责读取管理层汇总、生成决策方案、下发战略任务

`langgraph-supervisor-py` 可以参考它的分层调度方式：

- 顶层 Supervisor 不直接处理所有细节，而是把任务交给合适的下级 Supervisor 或 Agent。
- 中间层 Supervisor 负责管理自己的子团队。
- Worker Agent 只处理自己职责范围内的具体任务。

这对防止 AutoMage-2 中 Agent 越权、职责混乱、信息过载很有参考价值。

### 2. Supervisor 调度模式

胡文涛要做的 Manager Agent 和 Executive Agent，本质上都带有 Supervisor 属性。

可以参考 `langgraph-supervisor-py` 的调度思路：

- Executive Agent 判断任务应该交给哪个 Manager Agent。
- Manager Agent 判断任务应该交给哪个 Staff Agent。
- Staff Agent 完成任务后，把结构化结果返回给 Manager Agent。
- Manager Agent 汇总后，再向 Executive Agent 返回部门级结果。

这种 Supervisor → Worker 的任务流，可以转化为 AutoMage-2 的：

- 决策下发
- 部门拆解
- 员工执行
- 日报回流
- 管理层汇总
- 老板侧决策

### 3. 多层 Supervisor 设计

这个项目明确支持多层层级结构：顶层 Supervisor 可以管理多个子 Supervisor。

这点很适合参考 AutoMage-2 的组织结构：

- Executive Agent 对应顶层 Supervisor
- Manager Agent 对应子 Supervisor
- Staff Agent 对应 Worker Agent

后续如果公司有多个部门，也可以扩展成：

- Executive Agent
  - 销售部 Manager Agent
    - 销售 Staff Agent 1
    - 销售 Staff Agent 2
  - 运营部 Manager Agent
    - 运营 Staff Agent 1
    - 运营 Staff Agent 2
  - 技术部 Manager Agent
    - 技术 Staff Agent 1
    - 技术 Staff Agent 2

这比所有 Agent 扁平互相通信更适合企业级组织结构。

### 4. 职责隔离与路由逻辑

AutoMage-2 中非常重要的一点是：不同层级 Agent 不能越权。

可以参考 LangGraph Supervisor 的路由思想：

- Supervisor 只负责判断下一步交给谁。
- Worker 只负责执行被分配的具体任务。
- 不同 Agent 的能力通过工具 / Skill 限定。
- 每个 Agent 的上下文只保留与自己职责相关的信息。

对应到 AutoMage-2：

- Staff Agent 只能提交自己的日报、读取自己的任务。
- Manager Agent 只能读取本部门员工日报、生成部门汇总、处理权限内任务。
- Executive Agent 只能读取全局汇总、做最终决策和战略下发。

这有助于实现权限边界、数据隔离和 prompt 边界。

### 5. Agent 间消息传递方式

`langgraph-supervisor-py` 的一个重要参考点是：Agent 间不是随意聊天，而是通过 Supervisor 进行有控制的状态流转。

AutoMage-2 也可以采用类似原则：

- Staff Agent 不直接找 Executive Agent。
- Staff Agent 的日报进入数据库。
- Manager Agent 从数据库读取所属部门数据。
- Executive Agent 读取 Manager Agent 汇总结果。
- 决策结果再通过任务队列反向下发。

这样可以避免 Agent 间直接通信过乱，也符合会议纪要中提到的数据库契约数据通信路径。

### 6. 对胡文涛模板开发的具体参考

胡文涛可以参考这个项目，设计三类 `agents.md`：

#### Executive Agent

重点参考顶层 Supervisor：

- 读取 Manager 汇总
- 判断公司级风险
- 生成 A/B 决策方案
- 下发战略任务
- 不直接处理一线细节

#### Manager Agent

重点参考子 Supervisor：

- 读取下属 Staff 数据
- 生成部门汇总
- 识别 top risks
- 决定任务分发对象
- 超权限问题上推给 Executive Agent

#### Staff Agent

重点参考 Worker Agent：

- 执行具体任务
- 提交结构化日报
- 获取自己的待办
- 不做跨部门决策
- 不访问他人数据

## 和 AutoMage-2 的区别

`langgraph-supervisor-py` 是一个通用的分层多 Agent 编排项目，重点是 Agent 之间如何调度、路由和协作。

AutoMage-2 是企业 AI 中枢系统，除了 Agent 编排，还需要：

- 数据库契约存储
- Schema 校验
- 飞书卡片交互
- Cron 定时日报
- 组织权限体系
- Dream 决策机制
- 后端 API 联调

因此，它适合作为三级 Agent 架构、Supervisor 路由、职责隔离和任务流转的参考，不适合直接照搬为完整产品。

## 建议优先参考的部分

- 多层 Supervisor 结构
- Supervisor → Worker 的任务分发方式
- 子 Supervisor 管理团队的方式
- 不同 Agent 的职责隔离方式
- Agent 状态和消息流转设计
- 如何避免所有 Agent 扁平通信
