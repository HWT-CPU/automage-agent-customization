# 项目参考 07：VoltAgent

## 项目链接

https://github.com/VoltAgent/voltagent

## 项目简介

`VoltAgent` 是一个开源 AI Agent 工程平台，提供 Agent、工具、记忆、RAG、工作流、MCP、Webhook、Schedule 等能力。

AutoMage-2 有大量事件驱动和定时触发需求，例如日报提醒、二次催填、部门汇总、异常预警、老板决策下发等。因此 VoltAgent 可以作为工作流、调度器和事件触发设计的参考。

## 可以作为我们项目开发参考的内容

### 1. Schedule 定时任务

AutoMage-2 的定时任务非常明确：

- Staff Agent 每日 18:00 发送日报卡片
- Staff Agent 每日 20:00 二次催填
- Manager Agent 每日 21:00 汇总部门日报
- Manager Agent 每日 22:00 异常预警（待确认）

VoltAgent 支持 schedule / trigger 类能力，可以参考它如何组织定时任务、触发 Agent 执行和管理任务状态。

### 2. Webhook 事件触发

AutoMage-2 需要处理飞书回调事件：

- 员工提交日报卡片
- 老板点击决策按钮
- 管理员确认异常
- 外部系统推送业务变量

VoltAgent 的 webhook 思路可以参考为 AutoMage-2 的事件入口设计。

### 3. Agent Workflow 编排

AutoMage-2 不是单轮问答，而是一个固定业务流程：

- 触发提醒
- 收集数据
- 校验 Schema
- 写入数据库
- 汇总部门数据
- 生成决策方案
- 等待老板确认
- 下发任务

VoltAgent 的工作流能力可以参考如何把多步流程拆成稳定节点，并在节点之间传递状态。

### 4. Tool / MCP 能力

AutoMage-2 的 Skill 可以参考 VoltAgent 的工具层设计：

- 工具统一注册
- 工具参数结构化
- 工具返回可被 Agent 理解
- 外部 MCP 工具可被接入
- 工具调用和 Agent 推理分离

这有助于胡文涛设计 Hermes-Agent 的 Skill 层。

### 5. 事件驱动架构参考

会议纪要中明确提到后端是事件驱动、读多写少，Agent 侧也应该配合事件驱动。

可参考 VoltAgent 的思路，把 AutoMage-2 事件分为：

- schedule event：定时日报、定时汇总
- webhook event：飞书按钮、表单提交
- api event：后端任务队列变化
- alert event：鉴权失败、日报缺失、数据格式错误

每类事件再路由到对应 Agent 或 Skill。

## 和 AutoMage-2 的区别

VoltAgent 是通用 Agent 工程平台，不专门面向企业三级组织，也不自带飞书日报和 AutoMage-2 的数据库契约。

AutoMage-2 需要额外实现：

- Staff / Manager / Executive 三层模板
- 企业组织权限
- 数据库 Schema
- 飞书卡片
- Dream 决策
- 后端接口联调

因此 VoltAgent 适合作为 Cron、Webhook、工作流、事件触发和工具层工程化的参考。

## 建议优先参考的部分

- Schedule 定时任务设计
- Webhook 事件入口设计
- 工作流节点拆分方式
- Tool / MCP 工具层组织
- 事件驱动 Agent 执行方式
