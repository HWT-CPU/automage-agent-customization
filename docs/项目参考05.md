# 项目参考 05：mem0

## 项目链接

https://github.com/mem0ai/mem0

## 项目简介

`mem0` 是一个开源的 AI Agent 记忆层项目，定位是为 Agent 提供长期记忆、个性化上下文和可检索的历史信息管理。

AutoMage-2 中有 Dream 机制和每日 memory 整理需求，需要把员工日报、部门汇总、历史决策、任务反馈沉淀成可复用的组织记忆。因此 `mem0` 可以作为记忆层和 Dream 机制的重点参考。

## 可以作为我们项目开发参考的内容

### 1. 长期记忆层设计

AutoMage-2 不只是每天生成一次日报，还需要形成长期组织记忆：

- 员工历史表现
- 部门风险趋势
- 历史决策记录
- 老板偏好
- 任务完成情况
- 经常出现的问题

`mem0` 可以参考它如何把对话、事件和事实整理成长期可检索记忆。

### 2. Dream 机制参考

AutoMage-2 的 Dream 机制目标是：

- 读取每日 Staff 日报
- 读取 Manager 汇总
- 整理冗余信息
- 识别风险和模式
- 为 Executive Agent 生成决策输入

`mem0` 可以帮助参考：

- 记忆如何写入
- 记忆如何更新
- 记忆如何召回
- 如何区分短期上下文和长期记忆
- 如何避免重复记忆污染

### 3. 用户 / 部门 / 公司三级记忆

AutoMage-2 可以参考 mem0 的记忆组织方式，设计三级记忆空间：

- 员工级记忆：个人职责、历史任务、日报习惯、工作瓶颈
- 部门级记忆：部门目标、风险趋势、成员表现、资源使用
- 公司级记忆：战略目标、重大决策、ROI 偏好、跨部门矛盾

这和 AutoMage-2 的三级 Agent 架构相匹配。

### 4. 记忆召回与决策支持

Executive Agent 在生成 A/B 决策方案时，不能只看当天数据，还应该参考历史上下文。

可以参考 mem0 的记忆召回思想：

- 根据当前问题召回相关历史决策
- 根据部门名称召回部门过往风险
- 根据员工或任务召回历史执行记录
- 根据业务目标召回相关 KPI 和 ROI 经验

这可以提升 Dream 决策质量。

### 5. 与 Skill 层的结合

胡文涛负责的 Agent Skill 可以预留记忆接口，例如：

- `write_daily_memory`
- `retrieve_user_memory`
- `retrieve_department_memory`
- `retrieve_decision_history`
- `summarize_memory_for_dream`

即使 MVP 阶段不完整实现，也可以在模板里预留能力边界。

## 和 AutoMage-2 的区别

`mem0` 是通用 Agent 记忆层，不负责组织架构、飞书交互、数据库日报表、权限分级或任务队列。

AutoMage-2 需要自己实现：

- 组织结构权限
- 员工日报 Schema
- 部门汇总 Schema
- Dream 决策流程
- 飞书触达
- 任务下发闭环

因此，`mem0` 适合作为 Dream 机制、长期记忆、用户个性化和组织知识沉淀的参考。

## 建议优先参考的部分

- 长期记忆写入策略
- 记忆去重和更新方式
- 记忆召回逻辑
- 用户级记忆空间设计
- 历史决策辅助当前推理的方式
