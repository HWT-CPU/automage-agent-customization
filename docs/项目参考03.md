# 项目参考 03：CrewAI

## 项目链接

https://github.com/crewAIInc/crewAI

## 项目简介

`CrewAI` 是一个开源多 Agent 编排框架，核心能力是用角色、任务、工具和流程来组织多个 Agent 协作。

它强调 role-playing agent，也就是每个 Agent 都有明确身份、目标、背景、可用工具和任务边界。这一点非常适合参考 AutoMage-2 中的三级 Agent 模板设计。

AutoMage-2 需要的结构是：

- Staff Agent：一线执行者
- Manager Agent：部门汇总和任务分发者
- Executive Agent：公司级决策者

CrewAI 的角色化 Agent 和任务编排方式，可以帮助我们把这三类 Agent 的职责、输入、输出和调用工具规范化。

## 可以作为我们项目开发参考的内容

### 1. 角色化 Agent 定义

胡文涛需要编写三类 `agents.md`：

- Staff Agent：金牌执行官
- Manager Agent：精干中层
- Executive Agent：首席战略官

CrewAI 可以参考它定义 Agent 的方式：

- role：Agent 的身份
- goal：Agent 的目标
- backstory：Agent 的背景和行为风格
- tools：Agent 能调用的工具
- constraints：Agent 的行为边界

这些可以转化成 AutoMage-2 的 `agents.md` 模板结构。

### 2. 任务 Task 编排方式

AutoMage-2 的任务链路是：

- 员工提交日报
- 部门汇总日报
- 识别风险和待审批项
- 老板生成决策方案
- 任务反向下发

CrewAI 的 Task 概念可以参考：

- 每个任务有明确输入
- 每个任务有期望输出
- 每个任务绑定负责 Agent
- 多个任务可以按顺序或流程执行

这对设计 `post_daily_report`、`generate_manager_schema`、`dream_decision_engine` 等 Skill 的输入输出很有帮助。

### 3. 工具 / Skill 机制

CrewAI 中 Agent 可以绑定 tools。

AutoMage-2 中胡文涛要实现的 Skill，本质上也类似 tools：

- `post_daily_report`
- `fetch_my_tasks`
- `analyze_team_reports`
- `generate_manager_schema`
- `delegate_task`
- `dream_decision_engine`
- `broadcast_strategy`

可以参考 CrewAI 的工具绑定方式，设计：

- 哪个 Agent 能调用哪些 Skill
- Skill 的参数格式
- Skill 的返回格式
- Skill 调用失败后的处理方式

### 4. Crew / 团队协作模型

CrewAI 的 Crew 可以理解为一个 Agent 团队。

AutoMage-2 可以借鉴这种思想，把一个部门建模为一个 Crew：

- 一个 Manager Agent
- 多个 Staff Agent
- 统一的部门目标
- 统一的日报汇总流程
- 统一的风险识别逻辑

公司级 Executive Agent 则可以管理多个部门 Crew 的汇总结果。

### 5. Flow 流程控制

CrewAI 的 Flows 可以参考为 AutoMage-2 的固定业务流程：

- 每日 18:00 日报提醒
- 每日 20:00 二次催填
- 每日 21:00 部门汇总
- Dream 机制生成决策方案
- 老板确认后下发任务

这有助于把 AutoMage-2 从单个 Agent 能力，提升为稳定的业务流程系统。

## 和 AutoMage-2 的区别

CrewAI 是通用多 Agent 编排框架，不自带 AutoMage-2 所需的组织数据库、飞书卡片、企业权限和日报 Schema。

AutoMage-2 还需要额外实现：

- 数据库契约存储
- Schema 校验
- 飞书 IM 交互
- 权限分层
- Cron 调度
- Dream 决策机制

因此 CrewAI 适合作为角色模板、任务编排、Skill 绑定和部门 Crew 组织方式的参考。

## 建议优先参考的部分

- Agent role / goal / backstory 定义方式
- Task 输入输出设计
- Tool / Skill 绑定方式
- 多 Agent 团队协作方式
- Flow 流程编排思想
