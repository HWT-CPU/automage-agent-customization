# 03 里程碑二主链路数据库支持度核查

## 核查方法

- 以 **public 实际表结构** 为准，对照里程碑二 Mock 闭环所需读写路径。
- 「关键字段」以是否存在可映射物理列或可用 **jsonb/meta/payload** 兜底为准。
- 索引缺口为启发式：高频关联键上若无二级索引则标「建议补充」。

### 1. Staff 日报提交

| 维度 | 结论 |
|------|------|
| 是否已有对应表 | 是 |
| 对应表名 | staff_reports, work_records |
| 关键字段是否足够 | 部分关键字段需通过 meta/jsonb 或扩展列映射 |
| 是否需要通过 meta/jsonb 兼容 | 建议：是 |
| 是否缺字段 | payload |
| 是否缺索引 | 按查询模式复核 |
| 是否影响里程碑二 Mock 闭环 | 可支撑（部分字段建议 jsonb/meta） |
| 是否影响里程碑三真实联调 | 可进入真实联调验证 |

> staff_reports / work_records 至少其一可写即视为可支撑 Mock 验收写入。
