# 06 真实数据库与里程碑一契约差异分析

> 本分析以当前 public 实际表为准，对照里程碑一「以 work_records / summaries / tasks 为主落点」的叙述与里程碑二 Mock 包自述。

## 1. staff_reports 与 work_records 的关系

| 判断 | 说明 |
|------|------|
| 并存 | 真实库同时存在 `staff_reports` 与 `work_records` / `work_record_items`。 |
| 建议 | **Staff 日报主落点**：以 `staff_reports` 作为「提交壳 + 聚合索引 + payload/meta」；`work_records` / `work_record_items` 承载里程碑一兼容的结构化行项（任务进展、工作条目拆分）。 |
| 风险 | 若双写不一致，Manager 读取会出现「看得到壳、看不到行项」或相反；需 API 层事务或明确主从。 |

## 2. manager_reports 与 summaries 的关系

| 判断 | 说明 |
|------|------|
| 并存 | `manager_reports` 与 `summaries`、`summary_source_links` 同时存在。 |
| 建议 | **Manager 汇总主落点**：`manager_reports` 面向部门管理视角（健康度、签名、经理决策等）；`summaries` 作为跨层归一与检索索引，`summary_source_links` 绑定来源 Staff/Work 记录。 |
| 风险 | Mock 中 `aggregated_summary`、`top_3_risks` 等若拆表，需约定哪张表为权威读源。 |

## 3. decision_records / decision_logs / agent_decision_logs

| 判断 | 说明 |
|------|------|
| 分工 | **Executive 决策主实体**：`decision_records`（卡片、选项、确认态、payload）；`decision_logs` 记状态迁移与外部可见审计；`agent_decision_logs` 记 Agent/Dream 侧推理与工具轨迹。 |
| 与 summaries.meta | 若决策仅一次且需强查询，优先 `decision_records`；跨版本归并可镜像摘要进 `summaries`（非必须）。 |

## 4. task_queue 与 tasks / task_assignments / task_updates

| 判断 | 说明 |
|------|------|
| 建议 | **草案/排队**：`task_queue`；**正式任务**：`tasks`；**分配**：`task_assignments`；**状态事件流**：`task_updates`（PATCH 多写此表 + 更新 tasks 当前态）。 |
| 风险 | 若队列与正式表无关联键或状态机未定义，会出现重复下发或幽灵任务。 |

## 5. agent_profiles / agent_sessions / external_identities

| 判断 | 说明 |
|------|------|
| 用途 | **Agent 初始化主落点**：`agent_profiles` 长期配置；`agent_sessions` 会话/节点实例；`external_identities` 绑定飞书等外部主体与内部 user 对齐。 |
| 与 agent/init | `POST /api/v1/agent/init` 应以上述三表为写入核心，并回传 `node_id` / `session_id`。 |

## 6. audit_logs 是否覆盖全链路

| 判断 | 说明 |
|------|------|
| 能力 | 存在 `audit_logs` 表即可做 API 级 append-only 记录；是否足够取决于是否**每条主链路写操作**都写入、payload 是否包含 request_id / actor / target_id。 |
| 缺口 | 若仅有表无统一写入规范，仍会出现「库可审计、运行不可审计」。 |

## 7. 关键关联字段完整性（抽样）

以下字段为联调高频关联键，**存在性**基于当前库结构探测：

| 字段模式 | 存在于表（示例） |
|----------|------------------|
| org_id | users, work_records, tasks |
| department_id / dept | agent_sessions, decision_records, department_members, incidents, summaries, tasks, work_records |
| user_id / assignee | agent_decision_logs, agent_profiles, agent_sessions, artifacts, department_members, external_identities, manager_reports, staff_reports, summaries, task_assignments, task_queue, user_roles |
| public_id | agent_profiles, artifacts, decision_records, departments, external_identities, form_templates, incidents, organizations, roles, summaries, tasks, users, work_records |
| source / summary 链接 | summary_source_links, artifact_links |
